## 阶段 A — 数据底座（阻塞长历史研究）

> 目标：`2026-07-01`～`2026-07-15`（可再拉长）每个交易日都有发布版本，且 smoke 全绿。

### A0. 同步交易日历（首次或扩区间）

```bash
.venv/bin/python -m qdata.jobs.sync_calendar \
  --start 2026-06-01 --end 2026-07-31
```

| 备注 | 说明 |
|------|------|
| 功能 | 写入 ClickHouse `trade_calendar`，回填/日批只处理真正交易日 |
| 何时 | 首次；或研究区间超出已有日历 |
| 成功 | 打印 upsert 行数；之后 `trading_days_between` 不再用工作日近似 |
| 可跳过 | 日历已覆盖目标区间时，后续加 `--skip-calendar` |

### A1. 生产全市场回填（续跑）

```bash
# 日历已有时推荐：
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15 --skip-calendar

# 需要顺带质检（更慢，验收前建议开一次）：
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15 --skip-calendar --run-quality
```

| 备注 | 说明 |
|------|------|
| 功能 | 固定生产主源拉取 `stock_basic` + 日线/`adj`/`suspend` → Loader →（可选质检）→ `dataset_version` 发布；Raw 够行数会复用，便于断点续跑 |
| 何时 | 网络恢复后立刻；已发布日会尽量跳过重拉 |
| 成功 | 区间内每个交易日有 `daily_bar` 发布；日志无大片 `failed` |
| 失败 | BaoStock login/超时 → 稍后同一命令重跑；勿改 `MAX_SYMBOLS>0` |
| 背景跑 | `nohup ... > /tmp/qdata_prod_backfill.log 2>&1 &` 然后 `tail -f` |

### A2. 发布验收（门禁）

```bash
.venv/bin/python -m qdata.jobs.validate_release \
  --start 2026-07-01 --end 2026-07-15
```

| 备注 | 说明 |
|------|------|
| 功能 | 检查 `dataset_version` **连续性** + 区间内每日 smoke（universe ≥ `PROD_MIN_UNIVERSE`） |
| 何时 | A1 跑完后 |
| 成功 | 打印 `ACCEPTANCE PASS`（或等价全绿文案） |
| 失败 | 缺某日 version → 对该日/区间再 `prod_backfill`；**不要**只重复 validate |

### A2b. 单日 / 软 M2 冒烟（随时可跑）

```bash
# 核心 + M2 软检查（无行业/因子只提示，不失败）
.venv/bin/python -m qdata.jobs.smoke --date 2026-07-01 --check-m2

# 生产口径（强制已发布 + 全市场下限；同样带 M2 软检）
.venv/bin/python -m qdata.jobs.smoke --date 2026-07-01 --prod

# 区间逐日
.venv/bin/python -m qdata.jobs.smoke --start 2026-07-01 --end 2026-07-15 --prod
```

| 备注 | 说明 |
|------|------|
| 功能 | 验 `security_master` / 当日 `daily_bar` / 发布标记 / `get_universe(ALL)` / `get_price`；`--check-m2` 再软检指数·行业·因子·权重和 |
| 何时 | 每次入库或回填后；你已对 `07-01` 跑过且 **SMOKE OK** |
| 成功 | `SMOKE OK`；软提示「industry/mom 空」不阻断 |
| 解读 | `get_universe(000300.SH)≈287` 等表示指数 SCD2 已可用 |

### A3. 安装日批 cron

```bash
# 可选：在 .env 配置 QDATA_ALERT_WEBHOOK=钉钉/企微 URL
bash scripts/install_daily_cron.sh

# 确认
crontab -l | grep qdata
```

| 备注 | 说明 |
|------|------|
| 功能 | 工作日 17:30 自动跑 `daily_run --post-m2 --with-basic`，日志进 `logs/daily_run.log` |
| 何时 | A1/A2 基本稳定后挂上；也可并行挂、靠 webhook 看失败 |
| 成功 | crontab 含 `qdata.jobs.daily_run` |

### A4. 手工日批（联调 / 补跑某日）

```bash
# 盘后「今天」（非交易日自动跳过 exit 0）
.venv/bin/python -m qdata.jobs.daily_run --post-m2 --with-basic

# 指定日；行业较慢时再加 --sync-industry
.venv/bin/python -m qdata.jobs.daily_run --date 2026-07-15 --post-m2 --with-basic
.venv/bin/python -m qdata.jobs.daily_run --date 2026-07-15 --post-m2 --with-basic --sync-industry

# M2 后置失败也要让日批失败时：
.venv/bin/python -m qdata.jobs.daily_run --date 2026-07-15 --post-m2 --with-basic --strict-m2
```

| 开关 | 功能 |
|------|------|
| `--with-basic` | 拉 `daily_basic`（换手因子 `turn_20` 需要） |
| `--with-finance` | 拉利润表/资产负债表/现金流（完整需 Tushare） |
| `--post-m2` | 发布后：指数 snapshot + 种子因子（默认失败仅告警） |
| `--sync-industry` | 另同步申万 L1（HTTP 多，较慢） |
| `--strict-m2` | 后置失败 → 日批整体失败 |
| 验收 A4 | **连续 5 个交易日** 自动/手工 `daily_run` 成功 |

## 阶段 B — 选股域（指数 / 行业）

> 可与 A 并行：即使只有 `07-01` 一天日线，也可先把指数/行业 PIT 补齐。

### B1. 指数成分

```bash
# 日常：当日/指定日快照（单期无法还原全部历史调出）
.venv/bin/python -m qdata.jobs.sync_index_member --date 2026-07-01

# 多期 Raw 积累后重建 SCD2
.venv/bin/python -m qdata.jobs.sync_index_member --mode rebuild

# 正式历史回放（需 Tushare ENABLED+TOKEN+积分；沪深300 映射 399300.SZ）
.venv/bin/python -m qdata.jobs.sync_index_member \
  --mode tushare --start 2020-01-01 --end 2026-07-15
```

| 备注 | 说明 |
|------|------|
| 功能 | 写入 `index_member(in_date,out_date)`，供 `get_universe('000300.SH', d)` 时点选股 |
| 何时 | 研究用指数域前；你这边 **B1 已基本完成**（smoke 已见 287/429/771） |
| 成功 | 打印各指数 PIT count；DataAPI 非空 |
| 注意 | **禁止**用「最新成分」冒充多年回测选股域 → 用 `tushare` 或多次 snapshot + `rebuild` |

### B2. 申万行业 L1（优先）

```bash
.venv/bin/python -m qdata.jobs.sync_industry_member \
  --date 2026-07-01 --levels sw_l1
```

| 备注 | 说明 |
|------|------|
| 功能 | 拉申万一级行业成分 → Raw → 属性 SCD2 → `industry_member`；`industry` 形如 `801010.SI\|农林牧渔` |
| 何时 | smoke 提示 `industry_member 空` 时立刻跑（**已于 2026-07-17 完成**） |
| 成功 | 再跑 smoke `--check-m2` 不再提示行业空；`get_industry(['600000.SH'], date, 'sw_l1')` 非空 |
| 耗时 | 约 31 个行业 HTTP；可用 `--sleep 0.2` 限速 |

### B3. （可选）申万 L1+L2

```bash
.venv/bin/python -m qdata.jobs.sync_industry_member --date 2026-07-01
# 或多期后：
.venv/bin/python -m qdata.jobs.sync_industry_member --mode rebuild
```

| 备注 | 说明 |
|------|------|
| 功能 | L2 约百级行业，更慢；`rebuild` 用湖内多期闭合「调行业」区间 |
| 何时 | L1 稳定后再上 |

## 阶段 C — 种子因子 + 回测读数

> 依赖：日线历史足够长。`mom_20` / `vol_20` 约需 **≥21 个交易日**；仅 `07-01` 一天时算因子多为空属预期。

### C1. 计算种子因子

```bash
# 单日（lookback 不足时大量空值）
.venv/bin/python -m qdata.jobs.compute_factors --date 2026-07-01

# 区间（回填拉长后推荐）
.venv/bin/python -m qdata.jobs.compute_factors \
  --start 2026-07-01 --end 2026-07-15 --version v1

# 只算部分因子
.venv/bin/python -m qdata.jobs.compute_factors \
  --date 2026-07-15 --factors mom_20,vol_20
```

| 备注 | 说明 |
|------|------|
| 功能 | 基于已入库日线（+ basic）写入 `factor_value`：`mom_20`/`mom_60`/`vol_20`/`turn_20` |
| 何时 | A1 有足够历史后；smoke 提示 `mom_20 空` 时 |
| 成功 | `load_factor('mom_20', d, d)` 非空行数合理 |
| 版本 | 重算换 `--version v2`，勿覆盖旧回测依赖 |
| `turn_20` | 需要当日有 `daily_basic`（日批加 `--with-basic`） |

### C2. 读数层抽检

```bash
.venv/bin/python - <<'PY'
from datetime import date
from qdata.api.data_api import DataAPI
from qdata.research.portfolio import RebalanceSpec, build_weight_series, day_panel

api = DataAPI(allow_unpublished=True)
d = date(2026, 7, 1)
panel = day_panel(api, d, universe="000905.SH", factor="mom_20",
                  filters=["st", "suspended", "listed_days<120"])
print("panel", len(panel))

w = build_weight_series(
    date(2026, 7, 1), date(2026, 7, 1),
    api=api,
    spec=RebalanceSpec(universe="000905.SH", factor="mom_20", top_n=50),
)
print("version", w.attrs.get("dataset_version"))
print(w.head())
if not w.empty:
    print("weight_sum", w.groupby("trade_date")["weight"].sum())
PY
```

| 备注 | 说明 |
|------|------|
| 功能 | 按日：`universe → price → factor → 目标权重`（**不下单**） |
| 成功 | 同日 `weight` 求和 ≈ 1；`attrs['dataset_version']` 有值（如 `2026-07-01`） |
| 失败 | panel 空 → 先保证指数成分 + 因子非空 |

### C3. 记录版本（人工）

回测/研究笔记写明：

- `dataset_version`（如 `2026-07-01`）
- `factor` 名与 `version`（如 `mom_20` / `v1`）
- 选股域：指数代码 + filters

## 推荐执行顺序（一张清单）

按依赖从左到右；括号内为**你当前状态建议**。

```text
[环境] .venv + .env 生产项
   ↓
[A0]  sync_calendar（日历不够时）
   ↓
[A1]  prod_backfill 07-01～15 --skip-calendar     ← 网络恢复后优先
   ↓
[A2]  validate_release 同区间
   ↓
[B2]  sync_industry_member --levels sw_l1          ← 可立刻做（smoke 已提示空）
   ↓
[A2b] smoke --date 2026-07-01 --check-m2           ← 确认行业软提示消失
   ↓
[C1]  compute_factors（等日线历史够长）
   ↓
[C2]  build_weight_series 抽检
   ↓
[A3]  install_daily_cron.sh
   ↓
[A4]  连续 5 个交易日 daily_run OK
```

**可并行：** B1（你已基本完成）可与 A1 并行；B2 不依赖长日线。
