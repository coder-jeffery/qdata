# 数据采集操作说明

本文说明 qdata 采集层（Fetcher）的安装、配置、日常拉取与排障。目标读者：本地联调与日常跑批人员。

---

## 1. 整体流程

```
配置数据源 → 拉取数据集写入 Raw（Parquet）→ Loader 清洗入库 ClickHouse
```

| 阶段 | 作用 | 典型命令 |
|------|------|----------|
| 采集 | 从外部源拉数，落盘 Raw | `python -m qdata.fetchers ...` |
| 入库 | Raw → ClickHouse | `python -m qdata.loaders.* --fetch` |

约定：

- Raw 路径：`data/data-lake/raw/<source>/<dataset>/dt=<YYYY-MM-DD>/data.parquet`
- 内部统一代码：`exchange_code`（如 `600000.SH`）
- **行情 ETL** 用 `get_fetcher` / `qdata.fetchers`
- **交易下单** 用 `get_broker`（EasyTrader / MiniQMT 交易端），不要写进 `QDATA_DATA_SOURCE`

## 2. 环境准备

在项目根目录 `qdata/` 下操作。

### 2.1 基础安装

```bash
bash scripts/dev_install.sh
cp .env.example .env
```

> 本仓库的 `.venv` 由 uv 创建时，可能没有 `pip` 命令，请优先用：
>
> ```bash
> .venv/bin/python -m pip install ...
> ```

### 2.2 可选数据源依赖

| 需求 | 安装命令 |
|------|----------|
| 一键（含 Tushare / Efinance / MootDX / EasyQuotation） | `.venv/bin/python -m pip install -e '.[sources]'` |
| 仅 Tushare | `.venv/bin/python -m pip install -e '.[tushare]'` |
| 仅 JoinQuant | `.venv/bin/python -m pip install -e '.[joinquant]'` |
| 仅 Efinance | `.venv/bin/python -m pip install -e '.[efinance]'` |
| 仅 MootDX/PyTDX | `.venv/bin/python -m pip install -e '.[mootdx]'` |
| 仅 EasyQuotation | `.venv/bin/python -m pip install -e '.[easyquotation]'` |
| ZVT | `.venv/bin/python -m pip install -e '.[zvt]'`（需自行完成 ZVT 数据下载） |
| EasyTrader | `.venv/bin/python -m pip install -e '.[easytrader]'` + 本机交易客户端 |
| MiniQMT | 本机安装 QMT，并把 `xtquant` 加入 Python 路径（一般不在纯 PyPI） |

使用 Tushare 需同时打开开关并配置 token（[tushare.pro](https://tushare.pro)）：

```bash
QDATA_TUSHARE_ENABLED=true
QDATA_TUSHARE_TOKEN=你的token
```

默认 `QDATA_TUSHARE_ENABLED=false`：不参与 `auto` 故障转移，且 `--source tushare` 会报错。

使用 JoinQuant（[聚宽](https://www.joinquant.com)）需安装并开启：

```bash
.venv/bin/python -m pip install -e '.[joinquant]'
# .env
QDATA_JOINQUANT_ENABLED=true
QDATA_JOINQUANT_USER=19901712802
QDATA_JOINQUANT_PASSWORD=123456Jcm
```

默认 `QDATA_JOINQUANT_ENABLED=false`：不参与 `auto` 链；别名 `--source jq` 等价于 `joinquant`。

查看已注册源与能力：

```bash
.venv/bin/python -m qdata.fetchers --list-sources
```

### 2.3 导入失败时

若出现 `No module named 'qdata'`：

```bash
.venv/bin/python scripts/ensure_import_path.py
# 或
./scripts/qdata.sh -m qdata.fetchers --list-sources
```

## 3. 配置项（`.env`）

环境变量前缀均为 `QDATA_`。

### 3.1 数据源选择（必看）

| 变量 | 含义 | 示例 |
|------|------|------|
| `QDATA_DATA_SOURCE` | 当前源：单一名 / `auto` / 逗号链 | `auto`、`tushare`、`baostock` |
| `QDATA_DATA_SOURCE_CHAIN` | `auto` 时的回退顺序 | `akshare,baostock,tushare,joinquant,efinance,mootdx` |
| `QDATA_AKSHARE_MAX_SYMBOLS` | `>0` 时各源只拉前 N 只（联调用） | `30`；生产必须 `0` |
| `QDATA_PROD_SOURCE` | 生产唯一主源（禁 auto） | `baostock` / `tushare` |
| `QDATA_PROD_MIN_UNIVERSE` | 生产 smoke / Raw 复用下限 | `500` |
| `QDATA_ALERT_WEBHOOK` | 日批钉钉/企微通知 | URL，可空 |
| `QDATA_JOINQUANT_ENABLED` | JoinQuant 总开关 | `true` / `false` |
| `QDATA_JOINQUANT_USER` / `PASSWORD` | 聚宽账号 | — |

推荐：

- **生产回填/日批**：`QDATA_PROD_SOURCE=baostock` + `MAX_SYMBOLS=0`（见 §9）
- **启用 JoinQuant**：`QDATA_JOINQUANT_ENABLED=true` + 账号密码，再设 `--source joinquant`
- **启用 Tushare**：`QDATA_TUSHARE_ENABLED=true` + token，再设 `QDATA_DATA_SOURCE=tushare` 或放进 auto 链
- **关闭 Tushare（默认）**：`QDATA_TUSHARE_ENABLED=false`，继续用 baostock 等
- **联调冒烟**：再加 `QDATA_AKSHARE_MAX_SYMBOLS=30`

### 3.2 分源可选配置

| 变量 | 用途 |
|------|------|
| `QDATA_TUSHARE_ENABLED` | Tushare 总开关，默认 `false`；`true` 才允许采集 |
| `QDATA_TUSHARE_TOKEN` | Tushare Pro token（开关打开后必填） |
| `QDATA_TUSHARE_RATE_LIMIT` | 每分钟调用上限，默认 `200`（按积分权限酌情调低） |
| `QDATA_JOINQUANT_ENABLED` | JoinQuant 总开关，默认 `false` |
| `QDATA_JOINQUANT_USER` / `PASSWORD` | 聚宽账号密码 |
| `QDATA_JOINQUANT_RATE_LIMIT` | 每分钟调用上限，默认 `60` |
| `QDATA_MOOTDX_SERVER` | 通达信主站 `host:port`，空则自选 |
| `QDATA_TDX_DIR` | 本地通达信目录（离线读 `.day`） |
| `QDATA_EASYQUOTATION_BACKEND` | `sina` / `tencent` / `qq` 等 |
| `QDATA_LAKE_ROOT` | Raw 根目录，默认 `data/data-lake` |

### 3.3 交易通道（非采集 ETL）

| 变量 | 用途 |
|------|------|
| `QDATA_BROKER` | `miniqmt` 或 `easytrader` |
| `QDATA_MINIQMT_PATH` / `QDATA_MINIQMT_ACCOUNT` | MiniQMT userdata 与资金账号 |
| `QDATA_EASYTRADER_*` | EasyTrader 券商类型、账号、客户端路径 |

## 4. 数据源一览

| 源名 | 类型 | 默认已装 | 主要能力 | 备注 |
|------|------|----------|----------|------|
| `akshare` | 历史 | 是 | 主数据、日线、复权、停牌、利润表 | 东财网络敏感 |
| `baostock` | 历史 | 是 | 主数据、日线、复权、停牌 | 免费兜底；`income` 暂不支持 |
| `tushare` | 历史 | 需 `[tushare]` + token | 主数据、日线、复权、停牌、daily_basic、三大表 | 正式跑批 / 财务备用；按积分计权 |
| `joinquant` | 历史 | 需 `[joinquant]` + 账号 | 主数据、日线、复权、**daily_basic / suspend / 三大表** | **推荐 extras 全量+增量**；别名 `jq` |
| `efinance` | 历史 | 需 `[efinance]` | 主数据、日线、复权 | 东财封装 |
| `mootdx` | 历史/实时 | 需 `[mootdx]` | 主数据、日线、复权、实时 | 通达信协议；可配服务器 |
| `easyquotation` | 实时 | 需 `[easyquotation]` | 实时快照、简易主数据 | **无历史日线** |
| `zvt` | 历史 | 需 `[zvt]` | 主数据、日线、复权 | 读本地 ZVT 库 |
| `miniqmt` | 历史/实时 | 需本机 QMT | 主数据、日线、复权、实时 | 行情用 Fetcher；下单用 Broker |
| `easytrader` | 交易 | 需客户端 | — | **不能**作为 `--source` 拉历史 |

`auto` 默认链：`akshare → baostock → tushare → joinquant → efinance → mootdx`  
一旦某备源成功，同一次拉取批次会**固定**该源，避免主数据与日线来自不同源导致代码对不上。  
`QDATA_TUSHARE_ENABLED=false`（默认）或未配 token 时，链路中的 `tushare` 会被跳过。  
`QDATA_JOINQUANT_ENABLED=false`（默认）时，链路中的 `joinquant` 会被跳过。

### 4.1 Tushare 接口映射

| 内部数据集 | Tushare Pro 接口 | 说明 |
|-----------|------------------|------|
| `stock_basic` | `stock_basic` | `ts_code` 即内部 `exchange_code` |
| `daily_bar` | `daily` | 按 `trade_date` 全市场；量：手→股，额：千元→元 |
| `adj_factor` | `adj_factor` | 按日全市场 |
| `daily_basic` | `daily_basic` | 换手/股本/市值/PE/PB |
| `suspend` | `suspend_d` | 当日停牌；无数据则空表 |
| `income` / `balancesheet` / `cashflow` | 同名接口 | 按 `ann_date` 公告日（PIT）→ `fin_statement` |

积分不够时接口会报错，可改用 `--source joinquant` / `baostock` 或提高账号积分。

### 4.2 JoinQuant 接口映射

| 内部数据集 | jqdatasdk | 说明 |
|-----------|-----------|------|
| `stock_basic` | `get_all_securities(['stock'])` | 代码 `600000.XSHG` → `600000.SH` |
| `daily_bar` | `get_price(..., fq=None)` | 按日批量；量已是股；额用 `money` |
| `adj_factor` | 不复权 close / 后复权 close | `fq=None` 与 `fq='post'` |
| `daily_basic` | `get_fundamentals(valuation)` | 换手/股本/市值/PE/PB；失败回退 `turnover_ratio` |
| `suspend` | `get_price(..., paused)` | `paused>0`；无停牌合法空表 |
| `income` / `balancesheet` / `cashflow` | `get_fundamentals` + `pubDate` | 按公告日增量/全量扫日 → `fin_statement` |

```bash
.venv/bin/python -m pip install -e '.[joinquant]'
.venv/bin/python -m qdata.fetchers --date 2026-07-15 --dataset daily_bar --source joinquant
# 联调可截断：QDATA_AKSHARE_MAX_SYMBOLS=30
```

> **daily_basic / suspend / fin_statement** 全量与增量命令见 **§9.6**。

## 5. 数据集说明

| 数据集 | 含义 | 入库 Loader |
|--------|------|-------------|
| `stock_basic` | 证券列表/主数据原料 | `security_master`（须先跑） |
| `daily_bar` | 不复权日线 OHLCV | `daily_bar` |
| `adj_factor` | 复权因子 | 随 `daily_bar` 合并 |
| `suspend` | 停牌 | `SuspendLoader` + 合并进 `daily_bar.is_suspended` |
| `daily_basic` | 换手/市值/估值 | `DailyBasicLoader` |
| `income` / `balancesheet` / `cashflow` | 三大表（PIT） | `FinStatementLoader` → `fin_statement` |
| `realtime_quote` | 盘中快照 | 一般只落 Raw，不进日线表 |

## 6. 日常操作

以下命令均在项目根目录执行，Python 用 `.venv/bin/python`（或 `./scripts/qdata.sh`）。

### 6.1 列出数据源

```bash
.venv/bin/python -m qdata.fetchers --list-sources
```

### 6.2 只采集到 Raw（不入库）

```bash
# 主数据
.venv/bin/python -m qdata.fetchers \
  --date 2026-07-15 --dataset stock_basic --source baostock

# 日线相关（可逗号多数据集）
.venv/bin/python -m qdata.fetchers \
  --date 2026-07-15 \
  --dataset daily_bar,adj_factor,suspend \
  --source baostock

# 实时快照
.venv/bin/python -m qdata.fetchers \
  --date 2026-07-15 --dataset realtime_quote --source easyquotation

# Tushare（需 QDATA_TUSHARE_ENABLED=true 且配置 TOKEN）
.venv/bin/python -m qdata.fetchers \
  --date 2026-07-15 \
  --dataset stock_basic,daily_bar,adj_factor,suspend \
  --source tushare

# 自动故障转移
.venv/bin/python -m qdata.fetchers \
  --date 2026-07-15 --dataset daily_bar --source auto
```

成功时终端类似：

```text
fetch[tushare] daily_bar 2026-07-15: 5000 rows -> .../raw/tushare/daily_bar/dt=2026-07-15/data.parquet
```

### 6.2.1 Tushare 采集并入库示例

```bash
# .env 至少包含：
# QDATA_TUSHARE_ENABLED=true
# QDATA_TUSHARE_TOKEN=xxxx
# QDATA_AKSHARE_MAX_SYMBOLS=30   # 可选，联调限票

.venv/bin/python -m pip install -e '.[tushare]'

.venv/bin/python -m qdata.loaders.security_master \
  --date 2026-07-15 --fetch --source tushare

.venv/bin/python -m qdata.loaders.daily_bar \
  --date 2026-07-15 --fetch --source tushare
```

### 6.3 采集并入库（推荐）

`--fetch` 会先拉 Raw，再 Loader 入库。

```bash
# 1）必须先主数据（exchange_code → security_id）
.venv/bin/python -m qdata.loaders.security_master \
  --date 2026-07-15 --fetch --source baostock

# 2）再日线
.venv/bin/python -m qdata.loaders.daily_bar \
  --date 2026-07-15 --fetch --source baostock
```

联调限制股票数时，在 `.env` 设置：

```bash
QDATA_AKSHARE_MAX_SYMBOLS=30
```

**重要**：`security_master` 与 `daily_bar` 必须用**同一数据源**（或同一次 `auto` 固定源），否则会出现「security_master 缺少映射」。

### 6.4 仅入库（Raw 已存在）

```bash
.venv/bin/python -m qdata.loaders.security_master --date 2026-07-15 --source baostock
.venv/bin/python -m qdata.loaders.daily_bar --date 2026-07-15 --source baostock
```

Loader 在 `auto` 下会按候选顺序在 `raw/akshare`、`raw/baostock` 等目录中查找已有分区。

### 6.5 推荐联调顺序（首次）

```bash
# .env
# QDATA_DATA_SOURCE=baostock
# QDATA_AKSHARE_MAX_SYMBOLS=30
# QDATA_CH_PASSWORD=quant123456   # 与本地 ClickHouse 一致

# 建表（若尚未建）
# clickhouse-client --multiquery < schema/clickhouse.sql

.venv/bin/python -m qdata.loaders.security_master --date 2026-07-15 --fetch --source baostock
.venv/bin/python -m qdata.loaders.daily_bar --date 2026-07-15 --fetch --source baostock
```

### 6.6 代理与网络

东财类源（AKShare / Efinance）易受代理影响。联调前可临时取消代理：

```bash
unset HTTP_PROXY HTTPS_PROXY ALL_PROXY http_proxy https_proxy all_proxy
```

东财不稳时直接：

```bash
--source baostock
```

MootDX 连不上主站时可指定：

```bash
# .env
QDATA_MOOTDX_SERVER=110.41.147.114:7709
# 或本机先：python -m mootdx bestip
```

## 8. 常见问题

### 8.1 `zsh: no such file or directory: .venv/bin/pip`

用模块方式安装：

```bash
.venv/bin/python -m pip install -e '.[sources]'
```

### 8.2 `security_master 缺少映射: ['600033.SH' ...]`

原因：主数据与日线股票池不一致（常见于 `MAX_SYMBOLS` + 不同源/不同截断规则）。

处理：用**同一 `--source`** 重拉主数据后再入库日线：

```bash
.venv/bin/python -m qdata.loaders.security_master --date 2026-07-15 --fetch --source baostock
.venv/bin/python -m qdata.loaders.daily_bar --date 2026-07-15 --fetch --source baostock
```

### 8.3 Raw 分区不存在

先 fetch，或 Loader 加 `--fetch`：

```bash
.venv/bin/python -m qdata.fetchers --date 2026-07-15 --dataset stock_basic --source baostock
```

### 8.4 `daily_bar` 成功 0 行

检查：

1. `--date` 是否为**交易日**
2. 网络 / 代理 / 数据源是否可用
3. `MAX_SYMBOLS` 是否过小或股票池为空

### 8.5 `easyquotation` 拉日线报错

预期行为：EasyQuotation 不做历史日线，请改用 `tushare` / `baostock` / `efinance` / `mootdx` / `akshare`。

### 8.6 Tushare 开关关闭

```text
RuntimeError: Tushare 渠道已关闭。请在 .env 设置 QDATA_TUSHARE_ENABLED=true ...
```

打开开关并配置 token。`auto` 模式下关闭时会从链路中排除，不影响其他源。

### 8.7 未配置 Tushare token

```text
RuntimeError: 未配置 Tushare token。请在 .env 设置 QDATA_TUSHARE_TOKEN=...
```

开关为 `true` 后仍需 token。`auto` 模式下未配置会跳过该源。

### 8.8 Tushare 权限 / 积分不足

接口返回权限错误时：检查账号积分是否覆盖 `daily` / `adj_factor` / `income` 等；临时改用 `--source baostock`，或设 `QDATA_TUSHARE_ENABLED=false`。

### 8.9 依赖未安装

```text
ImportError: 数据源 tushare 依赖未安装 ... pip install -e '.[tushare]'
```

按提示安装对应 extra 即可。

### 8.10 MiniQMT / EasyTrader

- MiniQMT：需本机 QMT 已登录，`xtquant` 可导入，并配置 `QDATA_MINIQMT_*`
- EasyTrader：需本机交易软件已登录，并配置 `QDATA_EASYTRADER_*`

## 9. 生产全市场回填与日批（M1.5）

生产链路详见 **[M1.5-生产回填验收.md](./M1.5-生产回填验收.md)**。本节给日常操作摘录。

### 9.1 生产配置（`.env`）

```bash
QDATA_PROD_SOURCE=baostock          # 或 tushare（需 ENABLED+TOKEN）
QDATA_PROD_MIN_UNIVERSE=500         # smoke / Raw 复用门槛；勿改成联调值如 30
QDATA_AKSHARE_MAX_SYMBOLS=0         # 生产必须全市场
# QDATA_ALERT_WEBHOOK=https://...   # 日批成功/失败通知（可选）
```

生产主源禁止 `auto` / 多源链。`prod_backfill` / `daily_run` 会调用 `apply_prod_mode()` 强制上述约束。

### 9.2 一键回填 + 验收

```bash
# 建议先短区间（约 1～2 周），再拉长到 ≥3 个月
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15

# 日历已同步时跳过（避免 BaoStock 日历接口卡住）
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15 --skip-calendar

# 仅验收（数据已发布）
.venv/bin/python -m qdata.jobs.validate_release \
  --start 2026-07-01 --end 2026-07-15
```

通过标志：`ACCEPTANCE PASS: dataset_version 连续 + smoke 全绿`。

流程：`生产模式` →（可选）`sync_calendar` → 全市场 `backfill`+`publish` → `validate_release`。

默认 **遇错即停**。中断后续跑同一命令即可：已完整的 Raw（`daily_bar`/`adj_factor`/`suspend`/`stock_basic`）会按 `PROD_MIN_UNIVERSE` 自动复用，不必重拉。

```bash
# 只补缺失日
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-02 --end 2026-07-02 --skip-calendar
```

进度日志：

```bash
# 若用 tee 跑批
tail -f /tmp/qdata_prod_backfill.log
```

### 9.3 日批无人值守

```bash
# 盘后跑「今天」（非交易日自动跳过，exit 0）
.venv/bin/python -m qdata.jobs.daily_run

# 推荐（与 cron / Airflow 对齐）：basic + extras 增量 + 指数/行业/因子
.venv/bin/python -m qdata.jobs.daily_run \
  --post-m2 --with-basic --with-extras \
  --sync-industry --industry-levels sw_l1,sw_l2

# 指定日 / 跳过质检（联调）
.venv/bin/python -m qdata.jobs.daily_run --date 2026-07-15 --skip-quality

# 安装 cron：工作日 17:30（同上推荐参数）
bash scripts/install_daily_cron.sh
# 日志：logs/daily_run.log
```

| 开关 | 作用 |
|------|------|
| `--with-basic` | 生产主源拉 `daily_basic`（`turn_20` 需要） |
| `--with-finance` | 生产主源同日拉三大表（需源支持；完整增量更推荐 `--with-extras`） |
| `--with-extras` | 发布后跑 `sync_market_extras` **水位增量**（默认源 `$QDATA_EXTRAS_SOURCE`/`joinquant`） |
| `--extras-only` | extras 子集：`basic,suspend,finance`（默认全开） |
| `--post-m2` | 发布后：指数 snapshot + 种子因子 + **软挂因子监控（A405）**（失败默认仅告警） |
| `--monitor-factors` / `--no-monitor-factors` | 强制开/关 A405；默认随 `--post-m2` 开启 |
| `--monitor-min-coverage` | 覆盖率告警阈值（默认 0.9） |
| `--sync-industry` | 同步申万行业（可用 `--industry-levels sw_l1,sw_l2`） |
| `--strict-m2` | M2/extras/监控后置失败则日批失败 |

A405 说明：监控落盘 `lake/factor_monitor/<date>/`；覆盖告警发 webhook 标题 `qdata factor_monitor WARN`，**不**把日批打成 `FAIL`（发布成功仍为 `daily_run OK`）。

冒烟软门禁：`python -m qdata.jobs.smoke --date <日> --check-m2`（`--prod` 自动开启）。

等价 Airflow DAG：`dags/daily_etl.py`（单 task 调用同一套 `run_daily` 参数，与 cron 对齐）。

### 9.4 质检 HARD

日批默认跑质检；区间生产回填加 `--run-quality`：

| 规则 | 级别 | 说明 |
|------|------|------|
| 行数完整性 | HARD | ≥ `PROD_MIN_UNIVERSE`；有昨量时相对昨量漂移 ≤5% |
| 涨跌停价正确性 | HARD | 抽样重算 `up_limit`/`down_limit` |
| 复权连续性 | HARD | 复权收益 \|r\|>21% 且非新股窗口 → 失败 |
| PIT 合理性 | HARD | `ann_date < report_date` 禁止 |
| 跨源抽样 | SOFT | 第二源未接时跳过 |

硬失败会阻断 `dataset_version` 发布。

### 9.5 常见失败

| 现象 | 处理 |
|------|------|
| `published=0` / missing versions | 先 `prod_backfill`，不要只跑 `validate_release` |
| `universe < min` | 检查 `MAX_SYMBOLS`、`PROD_MIN_UNIVERSE`；确认全市场 |
| BaoStock `login 失败` / 长时间无进度 | 网络问题；已带 login 重试+socket 超时；稍后 `--skip-calendar` 续跑 |
| 复权因子大量填 1.0 | `adj_factor` Raw 不完整；删残缺分区或等复用门槛触发重拉 |
| 日历同步卡住 | 用已有日历 + `--skip-calendar` |

### 9.6 daily_basic / suspend / fin_statement（多渠道全量 + 增量）

作业：`python -m qdata.jobs.sync_market_extras`  

用途：多渠道同步三类 extras（**保留 Tushare**；**推荐 JoinQuant** 做历史全量与按日增量）。

| 模式 | 含义 |
|------|------|
| `--mode full` | 必须指定 `--start`；扫完 `[start, end]` |
| `--mode incremental` | 从库内水位（`daily_basic` / `suspend` / `fin_statement` 最大日）次日拉到 `--end`（默认昨天）；无水位时默认回溯 30 日 |

行为备注：

- `daily_basic` / `suspend`：按**交易日**拉取并入库  
- `fin_statement`：按**日历日**扫公告日（`income` + `balancesheet` + `cashflow`）  
- 停牌日或无公告日返回空表为合法结果；`--source a,b` 故障转移时不会因此误切源  
- 前置：`.env` 开启对应源（`QDATA_JOINQUANT_ENABLED` / `QDATA_TUSHARE_ENABLED` + 凭证）

```bash
# ---------- 前置（JoinQuant）----------
.venv/bin/python -m pip install -e '.[joinquant]'
# .env:
#   QDATA_JOINQUANT_ENABLED=true
#   QDATA_JOINQUANT_USER=<11位手机号>
#   QDATA_JOINQUANT_PASSWORD=<密码>

# ---------- JoinQuant：历史全量（推荐）----------
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant \
  --mode full \
  --start 2024-01-01 \
  --end 2026-07-15

# ---------- JoinQuant：增量（日常）----------
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant \
  --mode incremental

# 增量指定结束日 / 强制起始日
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant \
  --mode incremental \
  --end 2026-07-15

.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant \
  --mode incremental \
  --start 2026-07-01 \
  --end 2026-07-15

# ---------- 只拉子集 ----------
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant --mode full \
  --start 2026-06-01 --end 2026-07-15 \
  --only basic,suspend

.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant --mode full \
  --start 2024-01-01 --end 2026-07-15 \
  --only finance

# ---------- Tushare（备用；需 token）----------
# .env: QDATA_TUSHARE_ENABLED=true  QDATA_TUSHARE_TOKEN=...
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source tushare \
  --mode full \
  --start 2026-07-01 \
  --end 2026-07-15

.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source tushare \
  --mode incremental

# ---------- 多源故障转移 ----------
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant,tushare \
  --mode incremental

# ---------- 其它开关 ----------
# 遇错即停（默认 continue-on-error）
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant --mode incremental --fail-fast

# 不写 dataset_version
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant --mode full --start 2026-07-01 --end 2026-07-15 --no-publish

# Raw 行数达标则跳过重拉（仅 basic/suspend）
.venv/bin/python -m qdata.jobs.sync_market_extras \
  --source joinquant --mode full --start 2026-07-01 --end 2026-07-15 \
  --reuse-min-rows 500
```

| 参数 | 说明 |
|------|------|
| `--source` | `joinquant` / `tushare` / `joinquant,tushare` 等（默认 `joinquant`） |
| `--mode` | `full` \| `incremental` |
| `--start` / `--end` | 日期 `YYYY-MM-DD`；incremental 时可省略 `--start` |
| `--only` | `basic,suspend,finance` 子集（默认三类全开） |
| `--fail-fast` | 遇错退出（默认尽量继续） |
| `--no-publish` | 不写 `dataset_version` |
| `--reuse-min-rows N` | Raw 复用门槛（basic/suspend） |
