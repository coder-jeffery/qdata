# 日频回测引擎（Backtest）— 架构与模块设计

> 状态：设计稿（尚未实现）  
> 日期：2026-07-18  
> 原则：复用 DataAPI + `research.portfolio` 权重层；引擎只做「撮合 → 持仓 → 净值 → 指标」；不下真金、不拉外网。  
> 关联：现有读数层 `src/qdata/research/portfolio.py`；交易层 `trading/` / `risk/` / `brokers/paper`（接口可对齐，运行时隔离）。

---

## 1. 目标与非目标

### 1.1 目标

| # | 目标 | 验收标准 |
|---|------|----------|
| G1 | 日频、可复现的多资产多空/多头回测 | 同配置两次 run 指标 bit-identical（浮点容差内） |
| G2 | 强制绑定数据版本 | 报告含 `dataset_version` + `factor_version` + 引擎 `engine_version` |
| G3 | A 股最小真实摩擦 | 整手、涨跌停不可成交、停牌跳过、佣金+印花税+滑点 |
| G4 | 与现有研究链打通 | 输入可直接吃 `build_weight_series` / `RebalanceSpec` |
| G5 | 产出可归档 | `run_id` + 明细 Parquet/CH 表，供 Dashboard / 对比 |

### 1.2 非目标（本阶段明确不做）

- 分钟/Tick 撮合、集合竞价微观结构  
- 融资融券完整券源、做空借券费率曲线  
- 组合优化求解器（仍在 `portfolio`；引擎只消费目标权重）  
- 真金下单、实时行情驱动  
- 完整归因（Brinson/Barra）——只留扩展点  
- 机器学习超参搜索平台

## 2. 默认交易假设（A 股日频）

以下为 **v1 冻结默认值**；均可在 `BacktestConfig` 覆盖，但改默认需升 `engine_version`。

| 项 | 默认 | 说明 |
|----|------|------|
| 调仓频率 | 按信号日（通常每日或每周） | 由权重序列的日期集合决定；无权重日 = 持仓不动 |
| 信号时点 | **T 日收盘后**可见因子/面板 | 与 DataAPI PIT / as-of 一致 |
| 成交时点 | **T+1 开盘**（`open`，复权口径见下） | 避免用 T 收盘成交造成前视；可改为 `next_close` |
| 价格复权 | 收益/净值用 **后复权**；涨跌停判断用 **不复权** 价与 `up_limit`/`down_limit` | 与质检 HARD 字段对齐 |
| 成交量约束 | v1 **不约束**成交量（假设充分流动性） | v1.1 可加 ADV% 上限 |
| 最小交易单位 | 买入向下取整到手（100）；卖出同 | 与 `risk` 整手规则一致 |
| 涨跌停 | 涨停不可买、跌停不可卖；部成 = 跳过该腿 | 用当日不复权 close 与限价比较 |
| 停牌 | 当日 suspend → 该腿不成交，权重差额进现金 | |
| ST / 次新 | 由上游 `get_universe` filters 处理 | 引擎不重复过滤，只信任权重输入 |
| 成本 | 佣金双边 `commission_rate`（默认 3bp，最低 5 元）；卖出印花税 `stamp_tax`（默认 5bp）；滑点 `slippage_bps`（默认 5bp，买卖各加） | |
| 初始资金 | `initial_cash`（默认 1e8） | |
| 基准 | 可选 `benchmark` 指数代码，用其日收益算超额 | 缺数据则跳过超额指标 |
| 空头 | v1 **仅多头**（权重 ≥ 0，和为 1） | 权重和 ≠ 1 时按比例归一或报错（可配） |

**时间线（默认 `execution=next_open`）：**

```
T 收盘后: 读因子 → 目标权重 W_T
T+1 开盘: 按 open 调仓（受涨跌停/停牌/整手约束）
T+1 收盘: 用 close 标记市值 → 记当日净值
```

| 层 | 职责 | 禁止 |
|----|------|------|
| DataGate | 统一取价/停牌/涨跌停/基准；强制 as-of | 直连 Fetcher |
| SignalPort | 目标权重时间序列（可外挂 portfolio） | 算因子 |

### 4.1 `config.py` — BacktestConfig

```python
@dataclass(frozen=True)
class BacktestConfig:
    start: date
    end: date
    initial_cash: float = 100_000_000.0
    execution: Literal["next_open", "next_close"] = "next_open"
    adjust: Adjust = "post"              # 净值计价
    lot_size: int = 100
    commission_rate: float = 0.0003
    commission_min: float = 5.0
    stamp_tax: float = 0.0005            # 仅卖出
    slippage_bps: float = 5.0
    weight_sum_tol: float = 1e-6
    renormalize_weights: bool = True     # 权重和≠1 时归一
    allow_partial_lot: bool = False      # False=整手下取整
    benchmark: str | None = "000905.SH"
    dataset_version: str | None = None   # None → DataAPI 最新 published
    engine_version: str = "bt-daily-0.1"
    run_name: str = ""
    persist: bool = True
```

校验：`start≤end`；费率 ≥0；`execution` 枚举；若指定 `dataset_version` 则 DataAPI 锁定 as-of。

### 4.2 `types.py` — 核心对象

| 类型 | 字段（要点） |
|------|----------------|
| `Order` | date, code, side, target_shares \| delta_shares, tag |
| `Fill` | date, code, side, shares, price, fee, rejected_reason\|None |
| `Position` | code → shares（不复权股数） |
| `LedgerSnapshot` | date, cash, market_value, nav, positions |
| `DailyResult` | date, nav, ret, turnover, n_fills, n_rejects, cash_ratio |
| `BacktestResult` | meta, equity_curve, fills, positions_panel, metrics |

所有金额 **CNY**；股数为整型。

### 4.3 `data_gate.py` — DataGate

**职责：** 一次回测内缓存，避免按日重复打 CH。

```python
class DataGate:
    def __init__(self, api: DataAPI): ...
    def trading_days(self, start, end) -> list[date]: ...
    def prices(self, codes, start, end, *, adjust) -> DataFrame:
        # columns: trade_date, exchange_code, open, close, up_limit, down_limit, volume
    def is_suspended(self, code, d) -> bool: ...
    def benchmark_returns(self, code, start, end) -> Series: ...
```

实现要点：

- 构造时 `DataAPI(version=config.dataset_version)`  
- 预取回测区间 **全市场或信号涉及代码并集** 的 OHLC+限价（按区间一次查询）  
- 停牌：读 `suspend` 表或 DataAPI 若已有封装；无则软降级为「无成交量且无行情」  

### 4.4 `signals.py` — 信号端口

```python
class WeightSignalProvider(Protocol):
    def weight_on(self, d: date) -> DataFrame:
        """空 = 当日无调仓；非空列: exchange_code, weight"""
    def meta(self) -> dict: ...
```

内置：

| 实现 | 用途 |
|------|------|
| `FromWeightFrame` | 吃已有 `build_weight_series` 输出 |
| `FromRebalanceSpec` | 内部调 `build_weight_series`（懒计算或预计算） |
| `FromTargetHoldings` | 目标股数（可选，v1.1） |

**调仓日定义：** `weight_on(d)` 非空的交易日 `d` 为信号日；实际成交在 `execution` 规则对应的成交日。

### 4.5 `rules.py` — FillRules

纯函数，便于单测：

| 函数 | 行为 |
|------|------|
| `round_lot(shares, lot, side)` | 买下取整、卖下取整到手 |
| `apply_slippage(price, side, bps)` | 买加卖减 |
| `can_buy(bar) / can_sell(bar)` | 停牌、涨停买、跌停卖 |
| `calc_fee(notional, side, cfg)` | 佣金（含最低）+ 印花税 |

涨跌停判定（建议）：

```
tradable_buy  = not suspended and close < up_limit * (1 - eps)
tradable_sell = not suspended and close > down_limit * (1 + eps)
```

开盘成交时：若只有 open/close，v1 用 **成交日 open** 做滑点基准，用 **成交日 close vs limit** 近似可买可卖（或要求 DataAPI 提供 open 时限价；若无则用 close 限价作保守代理，并在 meta 标记 `limit_proxy=close`）。

### 4.6 `broker.py` — BrokerSim

```python
class BrokerSim:
    def rebalance_to_weights(
        self,
        d_exec: date,
        target_weights: DataFrame,  # code, weight
        ledger: PortfolioLedger,
        bars: dict[str, Bar],
    ) -> list[Fill]:
        ...
```

算法（目标权重 → 目标市值 → 目标股数）：

1. 用执行价（open 或 close）估 `equity = cash + Σ shares * px`  
2. `target_value_i = weight_i * equity`  
3. `target_shares_i = round_lot(target_value_i / px_i)`  
4. `delta = target - current`；先卖后买（释放现金）  
5. 逐腿检查 `can_*`；失败记 `Fill(rejected_reason=...)`，不调整该腿  
6. 现金不足时：按买入清单等比缩减或跳过低优先（v1：**按因子权重从高到低买到现金耗尽**）  
7. 更新 ledger  

**拒单不抛异常**——写入 fills，保证回测可跑完。

### 4.7 `ledger.py` — PortfolioLedger

状态：

- `cash: float`  
- `shares: dict[str, int]`  
- 方法：`market_value(px)`, `nav(px)`, `apply_fill(fill)`, `snapshot(d, px)`  

市值计价默认用 **当日收盘价（后复权换算到可比，或统一用不复权价+分红忽略）**。  

**复权一致性（重要）：**

- 持仓股数按 **不复权成交股数** 记账  
- 若用后复权价算市值，必须用同一复权因子把历史持仓换到「复权价坐标系」，或 **全程用不复权价 + 忽略拆股现金分红（v1 简化）**  

**v1 推荐简化（可复现优先）：**

> 交易与市值均使用 **后复权 open/close**；不模拟分红再投资细节。涨跌停仍用 raw limit 字段判断可成交性。  
> 在 `BacktestResult.meta["price_model"] = "post_adjust_nav_raw_limits"` 标明。

### 4.8 `engine.py` — 日循环

伪代码：

```python
def run(self, signals: WeightSignalProvider) -> BacktestResult:
    days = gate.trading_days(start, end)
    ledger = PortfolioLedger(initial_cash)
    pending: dict[date, DataFrame] = {}  # exec_date -> weights
    equity_rows, fills_all = [], []

    for d in days:
        # 1) 若今日有待执行调仓 → BrokerSim
        if d in pending:
            fills = broker.rebalance_to_weights(d, pending.pop(d), ledger, bars[d])
            fills_all.extend(fills)

        # 2) 收盘估值
        snap = ledger.snapshot(d, closes[d])
        equity_rows.append(snap)

        # 3) 收盘后产生信号 → 登记到成交日
        w = signals.weight_on(d)
        if not w.empty:
            exec_d = next_trading_day(d) if execution=="next_open" else next_trading_day(d)
            # next_close: 仍是下一交易日收盘成交
            pending[exec_d] = w

    metrics = compute_metrics(equity_rows, benchmark_rets)
    return BacktestResult(...)
```

边界：

- 区间最后一天产生的信号若无下一成交日 → **丢弃并 warning**（不前视区间外）  
- 第一天无持仓时信号在次日首笔建仓  

### 4.9 `metrics.py`

| 指标 | 定义 |
|------|------|
| total_return | NAV_T / NAV_0 - 1 |
| ann_return | 按 242 交易日年化 |
| ann_vol | 日收益标准差 × √242 |
| sharpe | 默认 rf=0；(ann_ret)/ann_vol |
| max_drawdown | 净值峰谷 |
| calmar | ann_ret / \|mdd\| |
| turnover | 调仓日 Σ\|Δw\|/2 的均值（或成交额/双倍净值） |
| avg_cash | 现金占比均值 |
| hit_ratio | 日收益 >0 比例 |
| excess_* | 相对 benchmark（若有） |

输出：`metrics: dict[str, float]` + `equity_curve: DataFrame[trade_date, nav, ret, cash, mv]`。

## 5. 与现有模块的接口契约

| 现有模块 | 关系 |
|----------|------|
| `DataAPI` | 唯一行情/域/因子入口；回测只读 |
| `research.portfolio` | 信号生产；**不**改为依赖 backtest |
| `risk.check_order` | 规则语义对齐（整手/涨跌停/停牌）；回测 **不调用** Gateway，避免 Paper 状态污染 |
| `PaperBroker` | 共享 `CostModel`（`trading/cost.py`）；不复用 broker/ledger 实例 |
| `factor_value` / `dataset_version` | meta 强制记录 |

依赖方向（允许）：

```
portfolio → DataAPI
backtest  → DataAPI, portfolio(可选)
jobs      → backtest
```

禁止：`backtest → brokers/trading`（v1）。

## 7. 防前视清单（硬约束）

| 风险 | 防护 |
|------|------|
| T 日因子买 T 收盘 | 默认 `next_open` 成交 |
| 使用未发布未来日线 | DataAPI `version` as-of clamp |
| 财务 PIT 前视 | 信号侧已用 ann_date；引擎不重算财务 |
| 涨跌停用未来价 | 仅用成交日当日 bar |
| 区间外成交 | 无下一交易日则丢弃信号 |

单测：`test_backtest_no_lookahead.py` 构造「末日信号」断言无成交。

## 8. 性能设计

| 点 | 策略 |
|----|------|
| CH 往返 | DataGate 一次拉齐区间面板，内存 join |
| 股票池 | 仅信号出现过的代码 ∪ 持仓代码 |
| 日循环 | 纯 pandas/numpy，避免逐股 SQL |
| 规模目标 | 全市场 × 1 年日频 < 3 分钟（本机）；中证 500 × 3 月 < 30 秒 |

## 11. 风险与开放决策

| 决策点 | v1 选择 | 可选项 |
|--------|---------|--------|
| 成交时点 | next_open | next_close |
| 复权模型 | 后复权净值 + raw 限价 | 全 raw + 分红再投资 |
| 买不起时 | 高权重优先 | 等比缩放 |
| 结果存储 | Lake Parquet 优先 | 同步 CH |
| 与 Paper 对齐成本 | **共享 `CostModel`**（`trading/cost.py`） | 运行时仍隔离实例 |

若产品侧要改默认成交时点或复权模型，应在实现 BT-1 前冻结，并写入 `engine_version`。

## 13. 技术选型（与本仓库栈对齐）

总原则：**不引入第二套运行时**；算力与 IO 沿用 pandas / PyArrow / ClickHouse；回测内核自研，不绑重型第三方回测框架（A 股规则与版本契约更可控）。

### 13.1 选型总表

| 领域 | 选型 | 版本/形态 | 理由 | 明确不选 |
|------|------|-----------|------|----------|
| 语言 | **Python** | ≥3.12（现有） | 与 qdata / DataAPI / jobs 一体 | 另起 Rust/Java 引擎 |
| 数值与表 | **pandas + NumPy** | pandas≥2.2（现有） | 日频规模足够；与 portfolio 同构 | Polars（v1 双栈成本高；BT-4 可评估） |
| 列存 IO | **PyArrow Parquet** | ≥17（现有） | 与 data-lake 一致；run 归档标准 | 自研二进制、HDF5 |
| 研究库 | **ClickHouse** | 现有 `clickhouse-connect` | 只作 DataGate **读**行情/因子；结果 v1 不强制写 CH | 回测热路径每笔 SQL |
| 配置模型 | **`dataclasses(frozen=True)`** | 标准库 | 与 `RebalanceSpec` 一致；可序列化进 meta.json | 重依赖 Hydra/OmegaConf（v1） |
| 环境配置 | **pydantic-settings** | 现有 | 仅全局路径/CH；回测参数走 BacktestConfig | 把费率写进 .env（难复现） |
| CLI | **argparse** | 标准库 | 与 `jobs/*` 统一 | Typer/Click（无强需求不增依赖） |
| 作业入口 | **`python -m qdata.jobs.run_backtest`** | 现有 jobs 模式 | 可被 cron 偶发调用；非 Airflow 必挂 | 独立微服务 |
| 日志 | **logging** | 标准库 | 与 factors/jobs 一致 | loguru（无必要） |
| 测试 | **pytest** | ≥8（dev extra） | 规则/防前视单测；integration mark 测 CH | 无 |
| 内核框架 | **自研 `research.backtest`** | 本设计 | A 股涨跌停/整手/版本绑定要捏在手里 | Backtrader / Zipline / vn.py 策略引擎 |
| 绩效库 | **自研 metrics.py** | BT-1～3 | 指标少、定义要写进 meta | 强依赖 empyrical/quantstats |
| Tearsheet（可选） | **quantstats 或纯 HTML** | BT-4 optional-extra | 报告美化可后置 | 阻塞 BT-1 |
| 向量化加速 | **不引入** | — | 中证500×数年秒级可接受 | vectorbt / cuDF（过早） |
| 并行 | **单进程** | v1 | 复现优先；多 run 用进程外并行 | Ray/Dask 内嵌引擎 |
| 结果存储 v1 | **Lake Parquet + meta.json** | `data/data-lake/backtest/runs/` | 与 Raw 湖同哲学；易 diff | 一开始就上 CH 结果表 |
| 结果存储 v1.1 | **CH `backtest_*` 表** | 可选 | Dashboard / 多 run 对比查询 | 替代 Parquet（应双写或外挂） |
| API/UI | **暂无**；BT-4+ 再定 | — | 引擎先 CLI 证明 | 为回测先搭 FastAPI |
| 若做 BFF | **FastAPI** | 后置 | 异步无关，生态熟；只读查 run | Django |
| 若做研究前端 | **Vite + React** 或轻量 **Streamlit** | 后置二选一 | 内部工具 Streamlit 更快；产品化 React | 原生多页 PHP |
| 实验追踪 | **自研 run_id + meta.json** | v1 | 已有 dataset_version 契约 | MLflow/W&B（BT-4 可对接） |
| 成本/风控语义 | **对齐 `risk` 常量，代码不 import Gateway** | v1 | 避免 Paper 状态耦合 | 回测直接调 TradingGateway |

### 13.2 分层选型说明

**计算层（BrokerSim / Ledger / Metrics）**  
用 pandas DataFrame 做日截面、dict/Series 做持仓即可。热路径避免 `iterrows` 滥用：调仓日用向量化算目标股数，拒单列表再落明细。NumPy 仅用于收益/回撤向量。

**数据层（DataGate）**  
- 读：现有 `DataAPI` + 必要时 `db.query_df` 批量 SQL（一次拉区间 OHLC）。  
- 写：v1 **只写 Lake**；不在日循环写 CH。  
- 缓存：进程内 `dict` / 预切好的 DataFrame，不用 Redis。

**配置与复现**  
`BacktestConfig` → `asdict` → `meta.json`；`engine_version` 字符串手维。不引入 Hydra 多配置树，避免「同一 run 找不到覆盖层」。

**第三方回测框架为何不用**  

| 框架 | 不用的原因 |
|------|------------|
| Backtrader | 事件模型重；A 股涨跌停/整手要大量 hack；与 DataAPI 版本契约难绑 |
| Zipline / Zipline-reloaded | 美股假设深；维护与依赖重 |
| vectorbt | 强向量化，难表达拒单/部成与账本细节 |
| vn.py | 交易柜台导向，不是研究复现引擎 |

可在 BT-4 用 quantstats **只生成 HTML 报告**，不参与撮合。

### 13.3 依赖变更建议（`pyproject.toml`）

```toml
# 核心：零新增（pandas / pyarrow / clickhouse-connect 已够）

[project.optional-dependencies]
```

BT-1～BT-3 **禁止**为回测增加必选依赖。

### 13.4 与交付切片的对应

| 阶段 | 技术动作 |
|------|----------|
| BT-1 | 纯 Python + pandas + pytest；无新依赖 |
| BT-2 | PyArrow 写 run 目录；argparse CLI；DataAPI 只读 |
| BT-3 | 同栈；补 benchmark 序列计算 |
| BT-4 | 可选 quantstats；可选 CH DDL；可选 FastAPI 读 meta/equity |

### 13.5 决策冻结（选型侧）

1. 内核自研，不引入 Backtrader/Zipline/vectorbt。  
2. 结果 v1 只落 Lake Parquet。  
3. 不新增必选 PyPI 依赖。  
4. UI/MLflow 不进入回测 MVP 关键路径。

## 14. 参考入口（实现后）

```python
from datetime import date
from qdata.api.data_api import DataAPI
from qdata.research.portfolio import RebalanceSpec, build_weight_series
from qdata.research.backtest import BacktestConfig, BacktestEngine, FromWeightFrame

api = DataAPI(version="2026-07-15")
weights = build_weight_series(
    date(2026, 4, 1), date(2026, 7, 15),
    api=api,
    spec=RebalanceSpec(universe="000905.SH", factor="mom_20", top_n=50),
)
result = BacktestEngine(
    BacktestConfig(start=date(2026, 4, 1), end=date(2026, 7, 15),
                   dataset_version="2026-07-15"),
    api=api,
).run(FromWeightFrame(weights))
print(result.metrics, result.meta["run_id"])
```
