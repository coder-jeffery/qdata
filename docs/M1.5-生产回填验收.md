## 1. 配置

`.env`：

```bash
QDATA_PROD_SOURCE=baostock          # 或 tushare（需 ENABLED+TOKEN）
QDATA_PROD_MIN_UNIVERSE=500         # smoke / Raw 复用门槛（勿用联调值 30）
QDATA_AKSHARE_MAX_SYMBOLS=0         # 生产必须 0；prod 作业也会强制清零
# QDATA_ALERT_WEBHOOK=https://...   # 日批 webhook（可选）

# 若主源为 tushare：
# QDATA_TUSHARE_ENABLED=true
# QDATA_TUSHARE_TOKEN=...
```

规则：

- 生产主源 **禁止** `auto` / 逗号链  
- 允许值：`baostock` | `tushare`  
- `prod_backfill` / `daily_run` 会调用 `apply_prod_mode()` 固化上述约束

## 2. 一键生产回填 + 验收

```bash
# 建议先小区间验证门禁（例如 5～11 个交易日），再拉长到 ≥3 个月
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15

# 日历已有时跳过同步（避免网络卡住）
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15 --skip-calendar

# Tushare 主源
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-04-01 --end 2026-07-15 --source tushare

# 开启质检 HARD（日批默认开启；区间回填默认跳过以免误伤）
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15 --skip-calendar --run-quality
```

流程：`生产模式` → `sync_calendar`（可跳过）→ 全市场 `backfill`+`publish` → `validate_release`。

- 默认 **遇错即停**（保证版本连续）  
- **Raw 复用**：`daily_bar`/`adj_factor`/`stock_basic` 行数 ≥ `PROD_MIN_UNIVERSE` 则跳过重拉；`suspend` 分区存在即复用  
- BaoStock login 带重试 + socket 超时，避免无限挂死  

单日失败后续跑：

```bash
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-10 --end 2026-07-10 --skip-calendar
```

## 3. 仅验收（数据已存在）

```bash
.venv/bin/python -m qdata.jobs.validate_release \
  --start 2026-07-01 --end 2026-07-15

# 或
.venv/bin/python -m qdata.jobs.prod_backfill \
  --start 2026-07-01 --end 2026-07-15 --validate-only
```

成功输出末尾：

```text
ACCEPTANCE PASS: dataset_version 连续 + smoke 全绿
```

失败常见原因：

| 现象 | 处理 |
|------|------|
| missing versions / published=0 | 先跑 `prod_backfill`，不要只跑验收 |
| universe < min | 检查 `MAX_SYMBOLS` / `PROD_MIN_UNIVERSE`；确认全市场 |
| smoke 未发布 | 确认 publish 写入 `dataset_version` |
| 无交易日 | 先 `sync_calendar` |
| login / 无进度 | 稍后 `--skip-calendar` 续跑同一命令 |

## 5. 质检 HARD

| 规则 | 说明 |
|------|------|
| 行数完整性 | ≥ `PROD_MIN_UNIVERSE`；有昨量时相对昨量 ≤5% |
| 涨跌停价正确性 | 抽样重算比对入库 `up_limit`/`down_limit` |
| 复权连续性 | 排除新股无涨跌幅窗口后，\|复权收益\|>21% 阻断 |
| PIT 合理性 | 禁止 `ann_date < report_date` |

硬失败阻断 publish。

## 7. 与联调模式区别

| 项 | 联调 | 生产 M1.5 |
|----|------|-----------|
| 源 | `auto` / 多源 | 单一 `PROD_SOURCE` |
| 股票数 | `MAX_SYMBOLS=30` | 强制 `0`（全市场） |
| 发布 | 可选 | 必须 publish |
| smoke | 可不要求 published | 必须 published + 大 universe |
| 质检 | 常 `--skip-quality` | 日批默认开启 |
