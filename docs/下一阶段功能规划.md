# qdata 下一阶段功能落地规划与设计

> 现状：M1 采集 → Raw → Loader → ClickHouse → DataAPI 骨架已打通。  
> 原则：**先让研究读数正确、稳定、可复现，再谈因子与交易。**

交互版看板见 Cursor Canvas：`qdata-next-phase-roadmap`。

---

## 1. 目标分层

| 阶段 | 名称 | 周期 | 目标 | 启动条件 |
|------|------|------|------|----------|
| **M1.5** | 数据可用闭环 | 1–2 周 | 研究端稳定取数，日批无人值守 | **立刻** |
| **M2** | 研究与因子 | 2–4 周 | 时点正确选股域 + 种子因子 | 日线发布连续稳定 |
| **M3** | 实盘预备 | 其后 | 实时行情 + 下单可演练 | 因子/回测读数可用 |

硬约束顺序：

```
全市场日线稳定发布
  → 质检门禁可信
  → 指数/行业选股域
  → 种子因子
  → 回测读数层
  → 实时 / 下单
```

## 3. M1.5 — 数据可用闭环（立刻做）

### 3.1 功能清单

| 优先级 | 功能 | 设计要点 | 完成标准 |
|--------|------|----------|----------|
| P0 | 生产主源固化 + 全市场回填 | 日批固定 **单一主源**（有积分用 `tushare`，否则 `baostock`）；`MAX_SYMBOLS=0`；回填 ≥1 年日线 | `dataset_version` 连续；`smoke` 全绿 |
| P0 | 日批无人值守 | Airflow/cron 跑 `daily_etl`；失败 webhook；每日刷新主数据 | 连续 5 个交易日自动成功 |
| P0 | 质检规则补全 | 涨跌停重算 HARD；复权跳变排除新股；跨源抽样 SOFT | 硬失败阻断 publish |
| P1 | 财务 PIT 加固 | `income` + `balancesheet` + `cashflow` 按 `ann_date`；更正追加不覆盖 | `get_fundamental` 抽检无前视 |

### 3.2 关键设计

**主源策略**

- 正式发布链路 **禁止跨源拼日**：`stock_basic` 与 `daily_bar` 必须同一 source。
- `auto` 仅作灾备排障，不作为生产默认（避免代码池漂移）。

**发布契约**

```
fetch → load → quality(HARD) → publish(dataset_version) → 研究可见
```

- DataAPI 默认锁定最新 `published` 版本。
- 回测报告必须记录 `version`（如 `2026-07-15`）。

**回填策略**

1. `sync_calendar` 覆盖研究区间  
2. `backfill` 日线（可分段）  
3. 财务按公告日增量  
4. 失败日记入列表，支持单日重跑

## 4. M2 — 研究与因子

### 4.1 功能清单

| 优先级 | 功能 | 设计要点 | 落表 |
|--------|------|----------|------|
| P0 | 指数成分区间表 | 时点正确；`get_universe(指数)` | `index_member` |
| P0 | 行业分类区间表 | 申万 L1/L2 历史变更；`get_industry` | `industry_member` |
| P0 | 种子因子引擎 | publish 后增量计算；版本化 | `factor_value` |
| P1 | DataAPI 研究契约固化 | 强制 `adjust`；过滤器；PIT；`list_versions` | — |
| P1 | 简易回测读数层 | 按日：universe → price → factor → 权重；不下单 | 只读 CH（`research/portfolio.py`） |

### 4.2 指数成分设计

```
index_member(index_code, security_id, in_date, out_date)
查询：in_date <= T AND out_date > T
```

- 数据源：优先 Tushare；权限不足时 AKShare，并标注置信度。
- **禁止**用「最新成分」回放历史选股域。
- 首批指数：`000300.SH` / `000905.SH` / `000852.SH`（沪深300 / 中证500 / 1000）。

### 4.3 因子引擎设计

```
已发布 daily_bar (+ daily_basic / fin)
        ↓
   compute_factors(trade_date)
        ↓
   factor_value(factor_name, version, trade_date, security_id, value)
```

**种子因子（先少后稳）**

| 因子 | 输入 | 说明 |
|------|------|------|
| `mom_20` / `mom_60` | 后复权收盘 | 动量 |
| `turn_20` | turnover | 换手 |
| `vol_20` | 收益标准差 | 波动 |
| `ep` / `bp` | 财务+市值 | 有 basic/fin 后再上 |

约定：

- 重算使用 **新 version**，不覆盖旧回测依赖。
- 空值率、覆盖率写入当日质检报告（SOFT）。

### 4.4 回测读数层（不做完整回测框架）

最小接口即可：

```python
for d in trading_days:
    uni = api.get_universe("000905.SH", d, filters=["st", "suspended", "listed_days<120"])
    px = api.get_price(uni, d, d, adjust="post")
    f = api.load_factor("mom_20", d, d)
    # → 生成目标权重 DataFrame（研究输出）
```

下单、滑点、撮合留到 M3。

## 7. 本阶段明确不做

- 再堆免费爬虫源（边际收益低）
- 向量 / embedding 平台（与行情主线无关）
- 未验证的自动实盘
- 复杂组合优化器
- 全市场分钟线（无明确策略需求前）
